# 算子实现方案（类型一致性）

## 1. 目标

在不要求数值/语义等价的前提下，仅保证 **变异前后类型家族一致**。算子只需给出 **替换后的新表达式**，不依赖原始表达式。

## 2. 接入点

- 注册入口：`mutator/operator_registry.py`
- 规划器：`mutator/mutation_planner.py`
- 执行器：`mutator/mutation_executor.py`

算子通过 `OperatorSpec + apply_fn` 实现：

- `OperatorSpec.supported_slot_types`：限制可作用的槽位类型
- `OperatorSpec.supported_type_families`：限制可作用的数据类型家族
- `apply_fn(slot)`：返回新的 sqlglot Expression，用于替换 `slot.node_ptr`

## 3. 类型家族与算子设计

### 3.1 Numeric
适用槽位：`literal`, `function_arg`, `aggregate_arg`, `binary_expr`

建议算子：
1. `num_const_zero`：替换为 `0`
2. `num_const_one`：替换为 `1`
3. `num_const_random`：替换为随机整数（例如 `[0, 100]`）
4. `num_const_pi`：替换为 `PI()`
5. `num_const_rand`：替换为 `RAND()`

排序键专用算子（避免 `ORDER BY 42` 被解释为列位置）：
1. `order_num_const_zero`：替换为 `0 + 0`
2. `order_num_const_one`：替换为 `1 + 0`
3. `order_num_const_42`：替换为 `42 + 0`
4. `order_num_const_pi`：替换为 `PI() + 0`

### 3.2 String
适用槽位：`literal`, `function_arg`, `binary_expr`

建议算子：
1. `str_const_empty`：替换为 `''`
2. `str_const_fixed`：替换为 `'sample_x'`
3. `str_const_fixed2`：替换为 `'sample_y'`
4. `str_const_uuid`：替换为 `UUID()`
5. `str_const_concat`：替换为 `CONCAT('a', 'b')`

### 3.3 Datetime
适用槽位：`literal`, `function_arg`, `binary_expr`

建议算子：
1. `dt_const_date`：替换为 `'2023-01-01'`
2. `dt_const_datetime`：替换为 `'2023-01-01 00:00:00'`
3. `dt_const_date2`：替换为 `'2023-12-31'`
4. `dt_current_date`：替换为 `CURRENT_DATE`
5. `dt_current_timestamp`：替换为 `CURRENT_TIMESTAMP`

### 3.4 Boolean
适用槽位：`predicate`, `binary_expr`

建议算子：
1. `bool_true`：替换为 `TRUE`
2. `bool_false`：替换为 `FALSE`
3. `bool_not_false`：替换为 `NOT FALSE`
4. `bool_compare_true`：替换为 `1 = 1`
5. `bool_compare_false`：替换为 `1 = 0`

### 3.5 JSON
适用槽位：`literal`, `function_arg`, `binary_expr`

建议算子：
1. `json_const_obj`：替换为 `'{"key":"value"}'`
2. `json_const_arr`：替换为 `'[1,2,3]'`
3. `json_object`：替换为 `JSON_OBJECT('k', 'v')`
4. `json_array`：替换为 `JSON_ARRAY(1, 2)`
5. `json_object2`：替换为 `JSON_OBJECT('a', 1)`

### 3.6 Null / Unknown
适用槽位：`literal`, `function_arg`

建议算子：
1. `null_literal`：替换为 `NULL`
2. `cast_to_string`：替换为 `CAST('1' AS CHAR)`
3. `cast_to_numeric`：替换为 `CAST('1' AS DOUBLE)`
4. `coalesce_default`：替换为 `COALESCE(NULL, <type-const>)`

## 4. 槽位约束建议

- `predicate`：仅允许 boolean 算子
- `order_key`：数值常量使用表达式形式（如 `42 + 0`），避免被解释为列位置
- `aggregate_arg`：按聚合函数语义限制（如 `SUM/AVG` 只允许 numeric）
- `binary_expr`：按操作符类别限制（比较/算术/字符串）
- `function_arg`：按函数参数类型族限制

## 5. 规划器策略建议

当前规划器只取 `candidates[0]`，若希望同一槽位尝试多个算子：

- 修改 `MutationPlanner.plan()` 返回多算子组合
- 或仅在 `plan_singletons()` 中对每个算子生成独立计划

## 6. 最小实现步骤

1. 在 `operator_registry.py` 中新增算子注册函数
2. 实现对应 `apply_fn` 并返回 sqlglot Expression
3. 用 `OperatorSpec` 指定 `slot_type` 和 `data_type_family`
